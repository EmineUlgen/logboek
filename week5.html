    Week 5

    HTML basis, GitHub upload & eerste keer werken met Netlify

    In week 5 leerde ik voor het eerst de basis van HTML en hoe je structuur opbouwt in een webpagina. Ik begon met het opruimen van mijn code, het aanpassen van tekst, kleuren en typografie, en het verbeteren van de lay-out zodat mijn logboek overzichtelijker werd. Kleine CSS-aanpassingen zoals padding, marges en lettergroottes maakten meteen een groot verschil.

    Daarna moest ik mijn project uploaden naar GitHub en koppelen aan Netlify. Dit was het moeilijkste gedeelte van deze week. Ik kreeg constant errors: GitHub herkende soms mijn bestanden niet, en Netlify gaf foutmeldingen waardoor mijn site niet wou laden. Ik wist toen nog niet hoe alles werkte en moest veel dingen opzoeken om te begrijpen wat ik fout deed.

    Na veel proberen, opnieuw beginnen en extra uitleg vragen, begreep ik eindelijk hoe de workflow werkt: HTML → GitHub → Netlify → online website. Nu vind ik het logisch, maar toen was het echt even zoeken en experimenteren.

    In Visual Studio Code beheerde ik alle HTML- en CSS-bestanden en probeerde ik de pagina’s overzichtelijk te maken. AI-hulp zorgde ervoor dat ik sneller foutjes kon opsporen, zoals ontbrekende tags of verkeerde klassen, waardoor ik beter begreep hoe HTML in elkaar zit.

    Het doel van deze week was om een volledig werkend en online logboek te hebben. Alle stappen van week 1 t.e.m. week 5 — schetsen, low-fi, mid-fi, Figma en HTML — komen samen. Ondanks de fouten bij GitHub en Netlify heb ik veel bijgeleerd over hoe je een website daadwerkelijk publiceert op het internet.

    Wat ik heb geleerd

    • De basis van HTML en hoe je een duidelijke structuur maakt.
    • Hoe je projecten op GitHub zet, ook al liep het vaak mis in het begin.
    • Hoe je Netlify gebruikt om een website online te zetten.
    • Dat foutmeldingen normaal zijn en deel uitmaken van programmeren.
    • Dat ik door veel proberen, googlen en hulp vragen uiteindelijk alles begrijp.

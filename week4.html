    Week 4

    Figma ontwerpen omzetten naar HTML & CSS in Visual Studio Code

    In week 4 begonnen we voor het eerst met het omzetten van onze Figma-ontwerpen naar HTML en CSS in Visual Studio Code. Dit was een grote stap, omdat ik tot nu toe enkel schetsen en digitale prototypes had gemaakt. Nu moest ik leren hoe die schermen er echt zouden uitzien in een browser.

    De leerkracht vroeg om onze Figma-schermen naar ChatGPT te sturen zodat AI een eerste versie van de HTML en CSS kon genereren. Dit hielp om te begrijpen hoe zo’n ontwerp vertaald wordt naar code, maar de resultaten waren zeker nog niet perfect: tekst stond scheef, elementen waren te groot of te klein en de layout klopte vaak niet. Toch was dit een waardevolle oefening om te leren hoe je structuur opbouwt in HTML.

    Ik merkte dat ik nog veel moeite had met begrijpen welke informatie precies op een treinscherm hoort, en hoe je die het best structureert. Mijn nieuwe versies van de schermen waren iets beter, maar nog steeds vrij basic. Het was vooral een periode van uitproberen, fouten maken en leren.

    Ik leerde hoe je containers maakt voor het overzichtsscherm, perronscherm en wagonscherm. Zelfs al was het nog niet perfect, ik begon te begrijpen hoe HTML elementen opdeelt en hoe CSS ze visueel op hun plaats zet.

    Door AI te gebruiken kon ik beter zien hoe flexbox, grids en typografie in code werken. Ik ontdekte ook hoe belangrijk het is om code te testen in de browser, want wat er op papier goed uitziet, ziet er in HTML soms totaal anders uit.

    Wat ik heb geleerd

    • Dat ontwerpen omzetten naar code veel complexer is dan ik dacht.
    • Dat HTML en CSS een duidelijke structuur nodig hebben om netjes te werken.
    • Dat AI kan helpen bij de basis, maar je zelf moet verbeteren en begrijpen.
    • Dat fouten deel zijn van het proces — mijn eerste versie was echt chaotisch, maar dat is oké.
    • Dat ik stap voor stap beter begon te snappen hoe webpagina’s zijn opgebouwd.
